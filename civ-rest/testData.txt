instructions:
To test the game-logic with the REST API, follow these steps:
* Start JBOSS AS server
* deploy civ-rest
* go to the index page of the deployment, typically http://localhost:8080/civ-rest-1.0/index.html
* in another tab or window open http://localhost:8080/civ-rest-1.0/rest/ksession/init to initialize the knowledge session
* one by one copy paste following JSON documents to the textarea on index.html page, after each document click "save to db" 
	and then overwrite the content in the textarea with another document. No other action required, the other textarea is NOT necessary
	for the basic scenario, the link "insert to ksession" is NOT used.
* With object dependent upon another ones in db it is necessary to assign them IDs generated by their insertion. The fields for generated IDs are
	denoted by commands like this: "SET_TILE_ID". As this field expects Long value, if not set properly the objects won't be inserted into db and
	an exception will occur.
* if needed scroll to the bottom of page where section "Perform Actions" resides after the section expands there are several categories
	of game objects such as player, unit, etc. This section serves as a browser through db data and also an interface to perform supported
	actions (move unit, build city, discover advance,...)
* It is necessary to click the load link of each section to get current data. The data change often, so to keep it up-to-date click load each
	time you want inspect them.
* Initially the turn regime is not started, to start it (after the objects are inserted) just access this url:
	 http://localhost:8080/civ-rest-1.0/rest/ksession/start
* To stop the turn regime access
	 http://localhost:8080/civ-rest-1.0/rest/ksession/stop
